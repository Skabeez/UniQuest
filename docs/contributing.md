# Contributing

Thanks for your interest in contributing to UniQuest!

## Code Style
- Follow Flutter/Dart style guides and `flutter_lints`.
- Keep changes minimal and focused; avoid unrelated refactors in single PRs.

## Branching & PRs
- Create feature branches from `main`.
- Submit PRs with:
  - What changed and why
  - Screenshots for UI changes
  - Testing notes

## Testing
- Run analyzer before pushing:
```bash
flutter analyze
```
- Run tests:
```bash
flutter test
```

## Commit Messages
- Use clear, imperative tone (e.g., "Fix loading page contrast").

## Reporting Issues
- Include steps to reproduce, expected vs actual, logs/screenshots, and environment details.
